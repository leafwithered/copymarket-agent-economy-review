Why it matters

AI agents can discover, buy, and consume useful work from other agents without a human sales cycle.

Solana role

Devnet settlement proof, escrow-gated delivery semantics, release logic, and refund/timeout path in the documented flow.

What to inspect

Buyer loop, seller bidding, structured JSON delivery, devnet proof, runbook, and tests.

Escrow State Machine

The project is not a chatbot. It is a small commerce protocol for paid agent work.

WANTBuyer defines job, budget, and criteria.
BIDSellers compete on value, scope, and price.
AWARDBuyer selects the best proposal.
ESCROW_REQUIREDSeller requires funded settlement before delivery.
DEPOSITEDBuyer funds the devnet settlement path.
DELIVEREDSeller returns structured JSON work.
RELEASEDPayment releases to seller, or timeout/refund path is used.

Buyer responsibilities

  • Publish a clear job and budget.
  • Select a seller from competing bids.
  • Fund settlement before expecting delivery.
  • Release payment or follow the refund path if delivery fails.

Seller and arbiter responsibilities

  • Seller bids and waits for funded terms before delivery.
  • Delivery is a structured package, not a chat transcript.
  • Escrow/arbiter path gates release and documents refund conditions.
  • The current public proof is devnet, not mainnet production readiness.

copyrescue
A structured landing-page rescue with hero headline, subheadline, CTA, section rewrites, conversion notes, and risk controls. It uses deterministic fallback output when no LLM key is available.

Path To Real Revenue
The current project is an honest devnet prototype. The next milestone is one real paid loop with stablecoin or mainnet settlement.

          

            
1. Productize copyrescue
Offer a fixed-scope landing-page rescue for crypto, SaaS, and AI-tool founders.

            
2. Price and collect
Quote a simple USDC amount or wallet invoice before delivery; keep the deliverable small.

            
3. Prove the paid loop
Publish order receipt, structured delivery JSON, and payment proof without exposing private keys.

            
4. Expand services
Reuse the same request, bid, escrow, delivery, release, and refund path for research, code review, compliance, and data jobs.

          

          
No real revenue is claimed yet. The project shows the settlement and delivery shape needed to move from devnet proof to a paid agent service.
